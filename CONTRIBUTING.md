# Contributing to GuardRail

Thanks for your interest in contributing! We welcome improvements, bug
fixes, tests, and documentation.

How to contribute
- Fork the repository and create a feature branch (e.g., `feature/xyz`).
- Run tests locally: `pytest`.
- Run linters/formatters: `ruff check .` and `black .` (we provide a pre-commit config).
- Open a pull request describing the change and its rationale. Include tests when possible.

Code style
- We use `ruff` for linting and `black` for formatting. Please run `ruff` and `black` before submitting.

Reporting security issues
- Do not open public issues for security vulnerabilities. Contact the maintainers privately via the repository's security policy.
